## 当前默认方案

| 项目 | 当前规则 |
|---|---|
| 原始素材优先级 | 用户提供的小说、大纲、剧本、人物参考图、场景图、风格图、音频、视频优先 |
| 生图 | 必须使用 `$imagegen` skill 的 built-in `image_gen`；静态图、参考图、关键帧、返修图都不走视频供应商、Web UI、OpenAI SDK 脚本或其他图片 provider |
| 关键帧参考绑定 | 只在提示词里写参考图文件名、路径或资产名不算绑定；人物、异兽、道具、线索进入关键帧前，必须把对应参考图实际加载/上传到当前 `$imagegen` 上下文，并在 source record 记录实际加载清单 |
| 音频 | Mimo；角色标准音用 `mimo-v2.5-tts-voicedesign`，正式对白用 `mimo-v2.5-tts-voiceclone` |
| 视频 | Seedance 多功能参考，默认 `seedance2.0`，不是 fast；视频上传/查询/下载必须通过本地 Dreamina/即梦 CLI |
| 视频单元 | 每段默认 15 秒，超过 15 秒硬失败；3 分钟单集通常约 12 段 |
| 分镜结构 | 每个 15 秒单元固定 4 个镜头/4 帧，镜头ID为 `SHOT01-SHOT04` |
| 连续性方案 | `keyframe_multiref_seedance`：上一段真实尾帧/SHOT01 + 有序关键帧 + 人物/场景/道具参考 + Mimo音频 |
| 执行工具 | `dreamina multimodal2video` 提交 Seedance 2.0 多功能参考视频；`dreamina query_result` 查询/下载结果 |
| 用户验收 | 视频提交前，用户只验收关键帧图片；脚本、提示词默认内部自检 |
| 关键帧验收页 | 优先使用 `$keyframe-review-gallery`；如果该 skill 不可用，运行 `scripts/build_keyframe_review_gallery.py` 生成本地 `index.html`、`status_reader.html` 和决策模板 |
| 输出组织 | 视频下载后保留原始文件，先用 Seedan 免费功能升至 `1080p`，再升至 `60fps`；当前成片按集建文件夹，按分镜段落顺序命名，并保留 `submit_id` 可追溯 |
| 上传图数 | Seedance `智能多帧/多模态参考` 最多 9 张图片；项目默认 7 图软上限。优先保留时序帧/上一段尾帧、主角、当前干净场景、可见剧情关键道具/痕迹；超出时先把未来场景改为文字承接，再省略可选风格/补充参考、可由时序帧恢复的配角参考和非关键道具，当前场景最后才省略 |
| 批量生产边界 | 批量生产不包含视频提交、排队、下载或抽尾帧；只做到上传包和检查报告 |
| 提交前核对 | 正式视频提交前必须生成并打开上传包核对页，展示真实上传顺序、省略引用、video audio、postmix-only audio 和最终提示词；用户确认后才可提交 |

## 开工方式

用户可以直接说：

```text
Use $ai-comic-drama 以我提供的小说、大纲、人物参考图、场景图和风格参考为准，先做素材整合、当前运行设置和大规模创作前确认表。
```

如果已经准备开正式样片：

```text
Use $ai-comic-drama 基于当前锁定素材，先做EP01的SEG001-SEG003：分镜脚本、关键帧提示词、关键帧验收包和Seedance上传包，不提交视频前先让我看关键帧。
```

如果要批量生产：

```text
Use $ai-comic-drama 按4+1制片厂结构准备批量生产：总控负责全季连续性、锁定资产、最终合并和待视频提交清单，四个agent分别负责EP01-05、EP06-10、EP11-15、EP16-20；本批量生产不提交、不排队、不下载视频。
```

## 第一次正式运行前

先创建或更新项目文件：

```text
00_项目设定/03_当前运行设置.md
大规模创作前确认表
用户原始素材整合清单
```

必须确认：

| 锚点 | 推荐状态 | 阻塞内容 |
|---|---|---|
| 小说/剧情基线 | 已确认可用 / 正式锁定 / 全季锁定 | 剧本、分镜、批量生产 |
| 集纲/节拍图 | 已确认可用 / 正式锁定 | 每集时长、钩子、节奏 |
| 主要角色参考图 | 正式锁定 / 全季锁定 | 角色关键帧、视频生成 |
| 角色锁点 | 正式锁定 / 全季锁定 | 跨段/跨集一致性 |
| 核心场景图 | 正式锁定 / 全季锁定 | 场景关键帧、空间连续性 |
| 干净场景参考/空间锚点 | 正式锁定 / 全季锁定 | 关键帧和视频空间锚定 |
| 道具/线索参考图 | 可见时必须正式锁定 | 正式关键帧和视频提交 |
| 视觉风格圣经 | 正式锁定 / 全季锁定 | 所有视觉生成 |
| 音频策略/角色标准音 | Mimo已确认 / 正式锁定 | 对白密集视频 |
| 视频链路/模型 | seedance2.0 | 付费视频提交 |
| 黄金样片/连续性测试 | 已通过 / 通过-有警告 | 批量上传包生产 |

图片和音频如果项目已经确认，不要反复询问。视频模型、成本、路线或样片结果改变时才重新确认。

## 批量生产边界

批量生产只包含：

- 素材整合、锁定资产检查、冲突/缺口报告；
- 分集大纲、钩子、每集时长和镜头密度检查；
- 15秒单元分镜脚本；
- 关键帧提示词、关键帧生成/验收包；
- 有序关键帧上传映射和视频提示词；
- Mimo音频生成表、音频映射、Seedance上传版音频文件；
- Seedance上传包、视频提示词、manifest；
- 自动检查报告和待视频提交清单。

批量生产不包含：

- 通过 `dreamina` CLI 提交 Seedance 视频；
- 任务排队/轮询；
- 视频下载；
- 从视频中提取尾帧；
- 生成下一段真实视频尾帧。

这些属于单独的 `视频提交/下载` 阶段。只有用户明确确认视频执行范围后，才可以逐段提交。

## 正式生产流程

1. 素材整合
   - 以用户提供文件为准，保留原始路径，整理项目副本。
   - 标记每个素材状态：`待确认`、`当前草案可用`、`已确认可用`、`小样锁定`、`正式锁定`、`全季锁定`、`需返修`、`缺失`、`明确跳过`。

2. 剧情和分集
   - 每集时长以已确认的 `EPISODE_SPEC` 为准；未确认时先停下来给规格菜单，不默认 3 分钟。
   - 每集必须有冷开场、15-30秒转折、中点升级、结尾钩子。
   - 每15秒作为一个单元分镜，每个单元固定4个镜头/4帧。

3. 15秒单元分镜
   - 每段默认 15 秒。
   - 每段写4行镜头，从 `SHOT01` 连续编号到 `SHOT04`。
   - 单元内时间码和每镜头时长依据分镜头脚本分配；每帧3-5秒，总时长仍为15秒。
   - 后续段 `SHOT01` 用来衔接上一段视频尾帧；如果是本集首段，则用于建立本集起始画面。
   - 每行必须写清剧情功能、景别/机位、画面动作、角色表演、音频口型、参考图、画面禁区和连续性要求。

4. 参考资产准备
   - 主要角色需要参考图和锁点。
   - 每个复用场景需要干净场景参考；空间源图可以保留在项目中，但不要把九宫格、地图或拼贴图作为正式视频上传图。
   - 可见且重要的道具、线索、痕迹必须有锁定参考图。
   - 道具上传参考必须是纯道具图，不带手、人物、场景、文字或拼贴。

5. 关键帧生成
   - 首段生成并验收本段 `SHOT01-SHOT04` 关键帧，最多4张。
   - 后续段 `SHOT01` 使用上一段真实尾帧，不重新生成；只生成并验收 `SHOT02-SHOT04`。
   - 每张关键帧只是一张 16:9 单张电影画面，不是分屏、故事板、参考图拼贴或角色设定图。
   - 关键帧引用分镜脚本行 + 人物提示词、人物参考图、干净场景参考、可见道具/线索参考。同一人物、同一道具不要同时出现两个以上。
   - 关键帧提示词必须按 `references/imagegen-staging.md` 的 reference-role 公式写：角色身份图、当前场景图、可见核心道具图、上一镜头/尾帧状态图各司其职；未来场景只写成下一镜头承接文本，除非角色已经进入该场景。
   - 所有关键帧、角色/场景/道具参考图和返修图必须通过 `$imagegen` / built-in `image_gen` 生成。本地文件名、路径和资产 ID 不会自动变成视觉输入；生成前必须先把本镜头所需锁定参考图实际加载/上传到当前 imagegen 上下文，并在 source record 写入实际加载清单。若只能使用文字描述，产物只能标记为 `DRAFT_TEXT_ONLY_NOT_FOR_REVIEW`，不得进入正式审片候选。
   - 每个新段或新角色首次进入关键帧时，先生成 1 张参考绑定验证图；确认脸型/发型/服装、异兽主体形态、核心道具外形与锁定参考一致后，再继续本段剩余镜头。验证失败的图片标记为 `CONSISTENCY_FAIL_NOT_PROMOTED`。

6. 用户关键帧验收
   - 视频提交前必须让用户验收当前视频范围内所有待提交关键帧。
   - 用户验收对象只有关键帧图片。
   - 分镜脚本行和关键帧提示词可以作为参考展示，但不要求用户批准文本。
   - 优先用 `$keyframe-review-gallery` 生成审片网页；如果该 skill 不可用，运行 `scripts/build_keyframe_review_gallery.py <段落包或manifest> --output <段落包>/keyframe_review_gallery`。
   - 审片网页只能收集 PASS/RETAKE/PENDING；必须把导出的结果另存为项目本地 `*_KEYFRAME_DECISIONS_CONFIRMED.json`，并同步更新 manifest 后，关键帧才算正式通过。

7. 有序关键帧上传映射
   - 取消正式多宫格故事板，不再对关键帧做拼贴或合板处理。
   - 首段：本段已通过关键帧按 `SHOT01-SHOT04` 直接上传，最多4张。
   - 后续段：上一段真实尾帧作为 `SHOT01` 和起始参考；本段已通过新关键帧按 `SHOT02-SHOT04` 选择上传，最多3张。
   - manifest 必须记录每张关键帧的 `shot_id`、路径、hash、用户验收状态和上传槽位。
   - 关键帧是节拍锚点，不允许生成成分屏、幻灯片或图片序列视频。

8. Seedance上传包
   - 包含上一段尾帧/起始关键帧、有序关键帧、人物参考、干净场景参考、必要道具/线索参考、Mimo音频、提示词、manifest。
   - Seedance `智能多帧/多模态参考` 最多上传 9 张图片；项目默认控制在 7 张。正式提交前必须按真实上传顺序数图；超出时按道具/痕迹、配角、场景的顺序有记录地省略，并在提示词里补偿连续性。
   - 视频提示词必须以这句结尾：`不要在视频画面中出现文字信息。`
   - 使用 `seedance2.0`、`16:9`、`15s`、`720p`，除非用户另行确认。
   - manifest 必须记录 `execution_provider: dreamina_cli`；上传包应能按真实上传顺序生成 `dreamina multimodal2video` 命令。
   - 正式提交前运行 `scripts/build_upload_package_viewer.py` 生成本地核对页，打开给用户确认；确认记录必须写回 manifest 或项目日志。

9. 自动检查
   - 上传包进入待视频提交清单前运行 skill 内置检查脚本：

```bash
~/.codex/skills/ai-comic-drama/scripts/check_keyframe_multiref_package.py <上传包文件夹> --out <检查报告.md>
```

检查重点：manifest、15秒规则、固定4个连续SHOT、上一段尾帧、有序关键帧、参考图、Mimo音频、提示词末句、尾帧记录。

10. 视频提交和下载（批量生产之外）
   - 视频上传、查询和下载必须通过本地 Dreamina/即梦 CLI。
   - 提交前运行 `dreamina -h`、`dreamina multimodal2video -h` 和 `dreamina user_credit`；如果登录失效，按 `dreamina login` 流程重新登录。
   - 默认提交命令为 `dreamina multimodal2video`，参数来自当前已确认上传包；不要手工加入未在 viewer 中确认的图片、音频或提示词。
   - 提交后记录 `submit_id`、命令、提示词、上传素材路径/hash、队列状态。
   - 查询和下载使用 `dreamina query_result --submit_id=<submit_id> --download_dir=<download_dir>`。
   - 下载完成后保留 Seedance 原始文件。
   - 使用 Seedan 免费功能按固定顺序处理：先把分辨率提升到 `1080p`，再把刷新率/帧率提升到 `60fps`。当前可检查、可交付版本以 `1080p/60fps` 文件为准。
   - 处理完成后按集归档，例如：

```text
assets/07_视频分段/SXXEXX_第XX集/SXXEXX_SEG001_v01.mp4
assets/07_视频分段/SXXEXX_第XX集/SXXEXX_SEG002_v01.mp4
```

   - 从 `1080p/60fps` 当前版本提取视频尾帧，作为下一段 `SHOT01` 和 `@图片1`。

11. 成片质检和返修
   - 检查首帧/尾帧、人物、场景、动作、音频、文字伪影、剧情价值、`1080p/60fps` 后处理和归档。
   - 返修要标明原因分类：脚本重写、关键帧重生、上传映射/提示词重建、视频重跑、音频重跑、只修manifest/归档。

## 关键帧提示词结构

正式关键帧提示词的唯一详细规则见 `references/imagegen-staging.md` 的 `Keyframe Prompt Continuity Rule`。本说明只保留摘要：

```text
镜头：[SXXEXX_SEGXXX_SHOTXX]，16:9单张电影关键帧。
引用图职责：图1=主角身份参考；图2=当前场景参考；图3=当前可见核心道具参考；图4=上一镜头/尾帧状态参考。每张图只按一个职责使用。
承接上一镜头：[上一动作状态、人物位置、道具状态、空间方向]
当前画面：[景别/机位、主体动作、身体部位、角色表情、道具状态]
为下一镜头承接：[下一动作的预备姿态或尾帧钩子]
限制：只出现一个主角；核心道具只出现一个；不生成分屏、拼贴、参考图、文字、UI、字幕、水印。
```

没有可见道具时删除道具引用行，不写“无道具”。返修时一次只改一个变量：参考图、动作、姿态、机位、道具状态或光线。

## 严禁事项

- 不用 `seedance2.0fast` 做测试或正式片。
- 不生成超过 15 秒的视频段。
- 不把 review contact sheet、角色设定图、空场景参考图当成关键帧或视频参考。
- 不用 Dreamina/即梦、Seedance、ChatGPT Web、OpenAI SDK 或项目自写脚本生成静态图；所有生成/返修图片必须走 `$imagegen` built-in `image_gen`。
- 不在用户未验收关键帧前提交视频。
- 不生成正式多宫格故事板；不把关键帧做成分屏、拼贴或图片序列视频。
- 不把同一角色或单体道具复制成多个。
- 不把全局 `.codex/generated_images` 缓存直接当项目资产库。
- 不覆盖锁定资产；返修必须新建版本。
- 不把真实 API key 写入 skill、manifest、报告或聊天。

## 批量协作

当小样和相邻段连续性测试通过后，20集季建议使用 `4+1制片厂结构`：

| 角色 | 范围 | 职责 |
|---|---|---|
| 总控 | 全季 | 锁定资产、连续性、最终合并、待视频提交清单、跨agent边界 |
| Agent A | EP01-EP05 | 分集剧本、分镜、资产包草稿 |
| Agent B | EP06-EP10 | 分集剧本、分镜、资产包草稿 |
| Agent C | EP11-EP15 | 分集剧本、分镜、资产包草稿 |
| Agent D | EP16-EP20 | 分集剧本、分镜、资产包草稿 |

共享文件只由总控更新。各 agent 写入自己的 `40_并行生产/Agent_X_EPxx-yy/` 文件夹。批量协作期间任何 agent 都不得提交、排队、下载视频或抽尾帧；只能把段落推进到 `READY_FOR_VIDEO_SUBMIT`。

## 本说明维护规则

更新以下内容时，必须同步更新本文件：

- 默认视频模型、分辨率、时长、平台路线；
- 关键帧验收门禁；
- 有序关键帧上传规则；
- 上传包结构和 manifest 字段；
- $imagegen 生图/Mimo/Seedance/即梦CLI 工具边界；
- 锁定等级和大规模创作前确认表；
- 自动检查脚本路径或检查项；
- 批量协作方式。
